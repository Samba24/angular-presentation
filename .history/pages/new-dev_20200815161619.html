1 - Le développement Web 2.0

Commençons d'abord par parler un peu de l'écosystème du développement Web en ce moment qui est en constante évolution. On assiste désormais à une ségrégation totale des différentes couches d'une application Web, traditionnellement la couche presentation, métier et persistance était tous gérer par une seule technologie.
On pouvait donc faire une application avec du PHP chargé de récupérer des données dans une base, faire le traitement nécessaire sur ces données puis présenter celles ci à l'utilisateur en utilisant peut être un moteur de rendu de vue.
L'avènement des frameworks JS qui apporte plus de réactivité niveau utilisateur, de plus de rapidité et surtout de flexibilité aux applications Web a amener un nouveau paradigme. La couche présentation est désormais totalement séparée de la couche métier, on parle de front-end et back-end. Cette architecture apporte une plus grande rapidité de développement pour les équipes et permet aux applications de ne plus être dépendante d'une seule technologie.

1.1 - Le back-end

Le back-end regroupe l'ensemble des technologies qui s'exécutent cote serveur et est chargé de gérer les transactions avec les bases de données. Il se charge de faire des traitements sur les données afin de les exposer sous un format de données bien défini, on parle d'API (Application Programming Interface).
Il existe des API RESTful ou SOAP, nous n'approfondirons pas ces concepts puisque ceux ci sort du cadre de notre exposé. Un article de comparaison est disponible à cette adresse pour les intéressés.

1.2 - Le front-end

Le front-end représente toutes les technologies qui sont utilisées afin de présenter des informations à l'utilisateur. Il correspond a la couche presentation et regroupe des technologies comme le HTML, le CSS, et le JS qui s'exécutent coté client c'est à dire au niveau du navigateur.
Les frameworks JS comme Angular, Vue, React et Amber sont retrouvés dans cette partie.
Le flux de données entre les deux applications est bidirectionnelle, l'application front envoie et reçoit des informations du back à l'aide de requêtes HTTP.
